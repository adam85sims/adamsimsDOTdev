Mock Tools

Simulate tools, APIs, and databases with configurable behavior.

Sentinel's mock layer replaces real tool calls with controllable fakes. This lets you test agent behavior without external dependencies.

MockTool

The core building block. A mock tool that returns canned responses:

Static response

from sentinel.env import MockTool

tool = MockTool("search", response={"results": ["Python is great"]})
result = tool(query="test")  # Returns {"results": ["Python is great"]}

Dynamic response

tool = MockTool(
    "calculator",
    response_fn=lambda expr: {"result": eval(expr)}
)
result = tool(expr="2 + 2")  # Returns {"result": 4}

Error injection

from sentinel.env import RateLimitError, TimeoutError

# Always fail
tool = MockTool("api", side_effect=RateLimitError("Too many requests"))

# Fail probabilistically
tool = MockTool("flaky", response="ok", error_probability=0.3)

Latency simulation

tool = MockTool("slow_api", response="data", latency_ms=500)

EnvironmentBuilder

Compose multiple tools into a test environment:

from sentinel.env import EnvironmentBuilder

env = (
    EnvironmentBuilder()
    .mock_tool("search", response={"results": []})
    .mock_tool("email", response="sent")
    .mock_tool("database", response={"users": []})
    .mock_api(base_url="https://api.example.com")
    .mock_database("users_db")
    .build()
)

# Access tools
result = env.tools["search"](query="test")

MockAPI

Simulate REST/GraphQL APIs with route matching:

env = (
    EnvironmentBuilder()
    .mock_api(
        base_url="https://api.example.com",
        routes={
            "GET /users": {"users": [{"id": 1}]},
            "POST /users": {"created": True},
        }
    )
    .build()
)

MockDatabase

In-memory database with CRUD operations:

env = (
    EnvironmentBuilder()
    .mock_database("users")
    .build()
)

# Operations available through the database
db = env.databases["users"]
db.insert({"id": 1, "name": "Alice"})
results = db.query("SELECT * FROM users WHERE id = 1")
💡 Chaos Integration All mock tools work with Sentinel's chaos injectors. Wrap a tool with ToolFailureInjector.wrap(tool) to add failure injection.
