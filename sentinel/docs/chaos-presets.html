Chaos Presets

Ready-made chaos configurations for common production failure scenarios.

Overview

Chaos presets are pre-configured lists of injectors that model real-world failure scenarios. Instead of manually combining injectors, you can import a preset and apply it to your test with a single line. Each preset is a list of injector instances ready to be added to a ChaosBudget or applied directly to tools.

📖 Import path from sentinel.chaos_presets import PRODUCTION_INCIDENT, DEPLOY_FRIDAY, TRAFFIC_SPIKE

Sentinel ships with 7 presets covering the most common production failure scenarios. You can also use the PRESETS registry for lookup by name.

Quick Usage

from sentinel.chaos_presets import PRODUCTION_INCIDENT
from sentinel.chaos import ChaosBudget

# Option 1: Apply presets directly to a budget
chaos = ChaosBudget(max_failures=10)
for injector in PRODUCTION_INCIDENT:
    chaos.add(injector)

# Option 2: Wrap individual tools
from sentinel.env import MockTool

db_tool = MockTool("database", response={"rows": 0})
for injector in PRODUCTION_INCIDENT:
    if hasattr(injector, 'wrap'):
        injector.wrap(db_tool)

# Option 3: Use the PRESETS registry
from sentinel.chaos_presets import PRESETS

preset = PRESETS["production_incident"]  # Same as PRODUCTION_INCIDENT

1. PRODUCTION_INCIDENT

Scenario: Database goes down, cascading to API and cache failures. The agent faces a multi-service outage with degraded context.

Injector Type Target Settings
ToolFailureInjector error database 80% failure rate, seed=42
ToolFailureInjector rate_limit api 40% failure rate, seed=42
ToolFailureInjector partial cache 30% failure rate, seed=42
ContextDegradation noise — 20% degradation rate

What it tests: Does the agent recognize that the database is the root cause? Does it fall back to cache (which returns partial data)? Does it degrade gracefully as context fills with error messages?

from sentinel.chaos_presets import PRODUCTION_INCIDENT

chaos = ChaosBudget(max_failures=8)
for injector in PRODUCTION_INCIDENT:
    chaos.add(injector)

# The agent should:
# 1. Detect database failures
# 2. Try alternative data sources
# 3. Report the issue rather than silently failing

2. DEPLOY_FRIDAY

Scenario: Friday deployment goes wrong. Deploy tool fails, health checks timeout, DNS intermittently fails, and error logs flood the context.

Injector Type Target Settings
ToolFailureInjector error deploy 90% failure rate, seed=42
ToolFailureInjector timeout health_check 60% failure rate, seed=42
ToolFailureInjector timeout dns 30% failure rate, seed=42
ContextDegradation noise — 30% degradation rate

What it tests: Does the agent detect that a deployment is failing? Does it roll back or halt? Can it distinguish between deployment failures and normal errors?

3. TRAFFIC_SPIKE

Scenario: Traffic spike hits. Rate limits trigger across services, database times out under load, cascading failures propagate, and memory pressure builds.

Injector Type Target Settings
ToolFailureInjector rate_limit api 50% failure rate, seed=42
ToolFailureInjector rate_limit search 40% failure rate, seed=42
ToolFailureInjector timeout database 30% failure rate, seed=42
CascadingFailures — — 60% cascade prob, max depth 2
MemoryPressure — — 8192 tokens, 70% threshold, FIFO eviction

What it tests: Does the agent implement backoff/retry? Does it handle rate limits gracefully? Can it survive cascading failures without crashing?

4. NETWORK_PARTITION

Scenario: Partial connectivity — external services and webhooks are isolated, but internal services (API ↔ database ↔ cache) remain connected. Context degrades during the partition.

Injector Type Settings
NetworkPartition — API↔DB↔Cache connected; external_api and webhook isolated; 100% partition
ContextDegradation truncation Default settings (10% rate)

What it tests: Does the agent detect that external services are unreachable? Does it use cached data? Does it report the connectivity issue?

from sentinel.chaos_presets import NETWORK_PARTITION

chaos = ChaosBudget(max_failures=5)
for injector in NETWORK_PARTITION:
    chaos.add(injector)

# The agent should:
# 1. Detect external service isolation
# 2. Fall back to internal services
# 3. Not hang waiting for unreachable services

5. TIME_TRAVEL

Scenario: Clock skew between services. The auth service has a 5-minute clock skew, causing token validation failures and API errors.

Injector Type Target Settings
ClockSkew — auth, token_refresh -300s skew (5 minutes behind)
ToolFailureInjector error api 40% failure rate, custom "Token expired" message

What it tests: Does the agent detect stale tokens? Can it refresh authentication? Does it handle timestamp mismatches without infinite retry loops?

6. MEMORY_LEAK

Scenario: Progressive context exhaustion. The context window fills up, triggering GC pauses and OOM kills. Search tool starts timing out under pressure.

Injector Type Target Settings
MemoryPressure — — 4096 tokens, 60% threshold, FIFO, 100ms GC, 10% OOM
ToolFailureInjector timeout search 20% failure rate, seed=42

What it tests: Does the agent handle context window exhaustion? Can it recover from an OOM kill (context reset to 0)? Does it prioritize keeping critical context?

from sentinel.chaos_presets import MEMORY_LEAK

chaos = ChaosBudget(max_failures=5)
for injector in MEMORY_LEAK:
    chaos.add(injector)

# The agent should:
# 1. Detect context pressure early
# 2. Consolidate or summarize to free space
# 3. Recover from OOM kills gracefully

7. COMPLETE_OUTAGE

Scenario: Everything is down. All tools fail at 100% probability, and full network isolation prevents any service communication.

Injector Type Target Settings
ToolFailureInjector error database 100% failure rate
ToolFailureInjector timeout api 100% failure rate
ToolFailureInjector error cache 100% failure rate
ToolFailureInjector timeout search 100% failure rate
NetworkPartition — — All services isolated, 100% partition

What it tests: The ultimate resilience test. The agent should recognize total failure, stop retrying, and report the outage clearly rather than entering an infinite retry loop or producing garbage output.

Customizing Presets

Presets are plain Python lists — you can modify them freely. Create a copy, add or remove injectors, or change parameters:

import copy
from sentinel.chaos_presets import PRODUCTION_INCIDENT
from sentinel.chaos import ToolFailureInjector, ChaosBudget

# Make a custom version with stricter database failures
custom_incident = copy.deepcopy(PRODUCTION_INCIDENT)

# Find and modify the database injector
for injector in custom_incident:
    if (isinstance(injector, ToolFailureInjector) 
            and injector.tool_name == "database"):
        injector.probability = 1.0  # 100% failure (was 80%)
        injector.failure_type = "timeout"  # Timeout instead of error

# Add an extra injector
custom_incident.append(ToolFailureInjector(
    tool_name="search",
    failure_type="rate_limit",
    probability=0.5,
    seed=42,
))

# Use it
chaos = ChaosBudget(max_failures=10)
for injector in custom_incident:
    chaos.add(injector)

Creating Your Own Presets

Presets are just lists of injectors. Create your own by following the same pattern:

from sentinel.chaos import (
    ToolFailureInjector,
    LLMFailureInjector,
    ContextDegradation,
    CascadingFailures,
    NetworkPartition,
    ClockSkew,
    MemoryPressure,
    ChaosBudget,
)

# Define a custom preset for "slow degradation" testing
SLOW_DEGRADATION = [
    # Gradual tool failures that increase over time
    ToolFailureInjector(
        tool_name="search",
        failure_type="timeout",
        probability=0.1,       # Low initial failure rate
        after_step=5,          # Start failing after 5 calls
        seed=42,
    ),
    # Context degrades slowly
    ContextDegradation(
        strategy="truncation",
        start_step=3,
        degradation_rate=0.05, # Very slow degradation
    ),
    # Memory pressure builds
    MemoryPressure(
        max_context_tokens=8192,
        pressure_threshold=0.9,  # High threshold — late pressure
        eviction_strategy="fifo",
    ),
]

# Register it in the PRESETS registry
from sentinel.chaos_presets import PRESETS
PRESETS["slow_degradation"] = SLOW_DEGRADATION

# Now you can use it by name
chaos = ChaosBudget(max_failures=5)
for injector in PRESETS["slow_degradation"]:
    chaos.add(injector)
💡 Preset design tips
  • Use seed on all injectors for reproducibility
  • Start with low probabilities and increase gradually
  • Combine tool failures with context degradation for realistic scenarios
  • Include at least one ToolFailureInjector per critical service

The PRESETS Registry

All presets are available via the PRESETS dict for lookup by name:

from sentinel.chaos_presets import PRESETS

# List all available presets
print(list(PRESETS.keys()))
# ['production_incident', 'deploy_friday', 'traffic_spike',
#  'network_partition', 'time_travel', 'memory_leak', 'complete_outage']

# Access by name
preset = PRESETS["traffic_spike"]

# Dynamic preset selection
preset_name = os.environ.get("CHAOS_PRESET", "production_incident")
chaos = ChaosBudget(max_failures=10)
for injector in PRESETS[preset_name]:
    chaos.add(injector)
