Agent Traces

The flight recorder for agent behavior.

What Is an AgentTrace?

AgentTrace is the central data structure in Sentinel. It captures every action your agent takes — tool calls, reasoning steps, state mutations, and errors — in an ordered, timestamped log. Assertions in Sentinel operate on the trace, not on output strings.

Think of it as a flight recorder: every decision, every API call, every failure gets logged so you can verify behavior after the fact.

from sentinel.models import AgentTrace

trace = AgentTrace()
# ... your agent runs against the trace ...
trace.finish()

print(f"Total steps: {trace.total_steps}")
print(f"Total tool calls: {trace.total_tool_calls}")
print(f"Duration: {trace.total_duration_ms:.1f}ms")

Data Structures

Step

A single atomic action in the agent's execution. Each step represents one unit of work — planning, calling a tool, reasoning, or producing output.

@dataclass
class Step:
    step_id: int                        # Unique step number
    action: StepAction                  # Type of action (see below)
    input: Any = None                   # What went into this step
    output: Any = None                  # What came out
    duration_ms: float = 0.0           # How long it took
    tool_calls: List[ToolCall] = []    # Tools invoked during this step
    error: Optional[Error] = None      # Error if this step failed

ToolCall

Records a single tool invocation by the agent. Captures the tool name, arguments, result, and timing.

@dataclass
class ToolCall:
    tool_name: str                      # Name of the tool (e.g. "search")
    arguments: Dict[str, Any]           # Args passed to the tool
    result: Any = None                  # Tool's return value
    duration_ms: float = 0.0           # Call duration
    error: Optional[str] = None        # Error message if call failed
    step_id: int = 0                   # Step this call belongs to

    @property
    def succeeded(self) -> bool:
        return self.error is None
ℹ️ The succeeded property A ToolCall succeeds if error is None. This is the primary way to check if a tool call worked. Use it in assertions like assert_tool_called which check the trace's tool call list.

StateChange

Records a mutation to agent memory or state. Tracks key-value changes to detect state inconsistency, staleness, and cross-session divergence.

@dataclass
class StateChange:
    key: str                  # State key (e.g. "current_order_id")
    old_value: Any = None     # Value before the change
    new_value: Any = None     # Value after the change
    step_id: int = 0          # Step where this change occurred

Error

Records an error encountered during agent execution.

@dataclass
class Error:
    message: str                              # Human-readable error message
    severity: ErrorSeverity = ErrorSeverity.MEDIUM
    step_id: int = 0
    recoverable: bool = True                  # Can the agent recover?
    timestamp: float = ...

Enums

StepAction

Identifies the type of action taken in a step:

ValueDescription
PLANAgent is planning its next move
TOOL_CALLAgent invoked a tool
REASONAgent is reasoning about data or context
RESPONDAgent produced output for the user
ERRORAgent encountered an error during this step

ErrorSeverity

ValueDescription
LOWMinor issue, agent can continue
MEDIUMNotable issue, may affect output quality
HIGHSerious error, likely impacts task completion
CRITICALFatal error, agent cannot continue

Building a Trace

Here's how to record agent behavior step by step:

from sentinel.models import AgentTrace, Step, StepAction, ToolCall, StateChange, Error, ErrorSeverity

trace = AgentTrace()

# Step 1: Agent plans
trace.add_step(Step(
    step_id=1,
    action=StepAction.PLAN,
    input="User asks about refund policy",
    output="Need to search for refund docs, then check user account",
))

# Step 2: Agent calls search tool
search_result = {"policy": "30-day refund window"}
trace.add_step(Step(
    step_id=2,
    action=StepAction.TOOL_CALL,
    input={"query": "refund policy"},
    output=search_result,
    duration_ms=45.2,
    tool_calls=[ToolCall(
        tool_name="search",
        arguments={"query": "refund policy"},
        result=search_result,
        duration_ms=45.2,
    )],
))

# Step 3: Agent calls database
db_result = {"user_id": 42, "order_count": 5}
trace.add_step(Step(
    step_id=3,
    action=StepAction.TOOL_CALL,
    input={"user_id": 42},
    output=db_result,
    duration_ms=12.0,
    tool_calls=[ToolCall(
        tool_name="database",
        arguments={"user_id": 42},
        result=db_result,
        duration_ms=12.0,
    )],
))

# Record state change
trace.add_state_change(StateChange(
    key="current_order_id",
    old_value=None,
    new_value="ORD-12345",
    step_id=3,
))

# Step 4: Agent responds
trace.add_step(Step(
    step_id=4,
    action=StepAction.RESPOND,
    output="Your refund policy: 30-day window. You have 5 prior orders.",
    duration_ms=8.0,
))

# Mark trace as complete
trace.finish()

Recording Tool Calls at Trace Level

You can also add tool calls directly to the trace (outside of steps) for quick recording:

# Direct tool call recording
trace.add_tool_call(ToolCall(
    tool_name="email",
    arguments={"to": "user@example.com", "subject": "Refund confirmation"},
    result={"sent": True},
    duration_ms=120.0,
))

Trace Properties

Once your trace is populated, use these properties to analyze the agent's behavior:

PropertyTypeDescription
total_stepsintNumber of steps recorded
total_tool_callsintNumber of tool calls (across all steps)
total_duration_msfloatWall-clock time from creation to finish()
tool_names_calledList[str]Unique tool names in order of first call
failed_tool_callsList[ToolCall]All tool calls where succeeded == False
 
# Query the trace
print(f"Steps: {trace.total_steps}")                    # 4
print(f"Tool calls: {trace.total_tool_calls}")          # 2
print(f"Duration: {trace.total_duration_ms:.1f}ms")     # 65.2ms
print(f"Tools: {trace.tool_names_called}")               # ["search", "database"]
print(f"Failed calls: {len(trace.failed_tool_calls)}")   # 0

# Get all calls to a specific tool
search_calls = trace.tool_calls_by_name("search")

Serialization

Traces can be serialized to dictionaries for reporting, storage, or JSON export:

# Convert to dict
trace_dict = trace.to_dict()
# Returns: {total_steps, total_tool_calls, total_duration_ms,
#           tool_names_called, failed_tool_calls_count, errors_count,
#           state_changes_count, metadata}
✅ Why traces matter Traces decouple what the agent did from what the agent said. You can assert that the agent called the right tools, handled errors gracefully, and didn't make unnecessary calls — even if the final output is non-deterministic.
