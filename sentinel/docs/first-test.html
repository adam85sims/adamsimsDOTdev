Your First Test

Write a complete behavioral test in 5 minutes.

Overview

This guide walks through writing a complete Sentinel test from scratch. You'll create a mock environment, run your agent against it, and verify its behavior with assertions. By the end you'll know two approaches: the pytest decorator (recommended) and the manual approach.

Using the @sentinel_test Decorator

The @sentinel_test decorator is the recommended way to write tests. It handles environment creation, trace injection, and chaos wiring automatically.

Step 1: Set Up the Environment

Use EnvironmentBuilder to create a controlled test environment with mock tools:

from sentinel.env import EnvironmentBuilder

env = (EnvironmentBuilder()
    .mock_tool("search", response={"results": ["Refund Policy", "Shipping Info"]})
    .mock_tool("database", response={"user_id": 42, "status": "active"})
    .build())

Step 2: Write the Test

Decorate your test function with @sentinel_test. The decorator injects a trace (AgentTrace) and env (Environment) into the test function:

import pytest
from sentinel.runner import sentinel_test
from sentinel.env import EnvironmentBuilder
from sentinel.assertions import assert_tool_called, assert_no_tool_errors

@sentinel_test(
    env=(EnvironmentBuilder()
        .mock_tool("search", response={"results": ["Refund Policy", "Shipping Info"]})
        .build()),
    task="Search for refund policy",
)
def test_search_agent(trace, env):
    # Your agent code here — run against the mock environment
    # For example, call the search tool and record behavior on the trace
    result = env.tools["search"](query="refund policy")

    # Add steps and tool calls to the trace
    from sentinel.models import Step, StepAction, ToolCall
    trace.add_step(Step(
        step_id=1,
        action=StepAction.TOOL_CALL,
        input={"query": "refund policy"},
        output=result,
        tool_calls=[ToolCall(
            tool_name="search",
            arguments={"query": "refund policy"},
            result=result,
            duration_ms=12,
        )],
    ))

    # Assert behaviors
    assert_tool_called(trace, "search")
    assert_no_tool_errors(trace)

Step 3: Run with pytest

pytest test_search_agent.py -v

You'll see pytest pick up the test normally, with the decorator handling all Sentinel plumbing behind the scenes.

ℹ️ The decorator's job @sentinel_test creates the AgentTrace, builds the environment (from EnvironmentBuilder or a pre-built Environment), and injects them as kwargs to your test. You just write assertions.

Adding Chaos Injection

Pass a ChaosBudget to the decorator to inject failures into your test environment automatically:

import pytest
from sentinel.runner import sentinel_test
from sentinel.env import EnvironmentBuilder
from sentinel.chaos import ChaosBudget, tool_failure
from sentinel.assertions import assert_tool_called

@sentinel_test(
    env=(EnvironmentBuilder()
        .mock_tool("search", response={"results": ["Refund Policy"]})
        .build()),
    chaos=ChaosBudget([
        tool_failure("search", failure_rate=0.3),
    ]),
    task="Search with 30% failure rate",
)
def test_search_handles_failures(trace, env):
    # The search tool now fails randomly ~30% of the time
    # Your agent should handle retries or graceful degradation
    try:
        result = env.tools["search"](query="refund policy")
    except Exception:
        pass  # Agent should handle this

    # At minimum, the tool was attempted
    assert True  # Add real assertions about recovery behavior
💡 Chaos tip Use chaos to verify your agent doesn't crash when tools fail. Good assertions for chaos tests check that the agent recovers or degrades gracefully rather than throwing uncaught exceptions.

The Manual Approach

If you prefer full control (or aren't using pytest), build the environment, create the trace, and run assertions manually:

from sentinel.env import EnvironmentBuilder
from sentinel.models import AgentTrace, Step, StepAction, ToolCall
from sentinel.assertions import assert_tool_called, assert_no_tool_errors

# 1. Build environment
env = (EnvironmentBuilder()
    .mock_tool("search", response={"results": ["Refund Policy"]})
    .mock_tool("database", response={"user_id": 42, "status": "active"})
    .build())

# 2. Create trace
trace = AgentTrace()

# 3. Run your agent (this is your actual agent code)
def my_agent(task, env, trace):
    """Simulate agent that searches and queries the database."""
    search_result = env.tools["search"](query=task)
    db_result = env.tools["database"](user_id=42)

    trace.add_step(Step(
        step_id=1,
        action=StepAction.TOOL_CALL,
        input={"query": task},
        output=search_result,
        tool_calls=[ToolCall(
            tool_name="search",
            arguments={"query": task},
            result=search_result,
            duration_ms=50,
        )],
    ))
    trace.add_step(Step(
        step_id=2,
        action=StepAction.TOOL_CALL,
        input={"user_id": 42},
        output=db_result,
        tool_calls=[ToolCall(
            tool_name="database",
            arguments={"user_id": 42},
            result=db_result,
            duration_ms=30,
        )],
    ))
    trace.finish()

my_agent("find refund policy", env, trace)

# 4. Assert behaviors
assert_tool_called(trace, "search")
assert_tool_called(trace, "database")
assert_no_tool_errors(trace)

# 5. Inspect trace
print(f"Steps: {trace.total_steps}")
print(f"Tool calls: {trace.total_tool_calls}")
print(f"Duration: {trace.total_duration_ms:.1f}ms")
print(f"Tools called: {trace.tool_names_called}")

Running with the CLI

You can also run tests via the Sentinel CLI instead of pytest:

# Run a specific scenario file
sentinel run --path my_scenario.yaml

# Run all discovered scenarios
sentinel run --all

# Run with verbose output
sentinel run --scenario refund-test --verbose

See YAML Scenarios for defining tests declaratively, or CI/CD Integration for running in pipelines.

Key Concepts Recap

  • EnvironmentBuilder — creates a controlled testing environment with mock tools, APIs, and databases.
  • AgentTrace — the flight recorder that captures every tool call, step, and error.
  • @sentinel_test — pytest decorator that wires everything together automatically.
  • Assertions — 20+ assertions to verify tool usage, error handling, governance compliance, and performance.
  • Chaos injection — randomly fail tools, degrade context, or simulate network issues to test resilience.
