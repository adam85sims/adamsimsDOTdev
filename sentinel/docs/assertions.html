Assertions

20+ behavioral assertions across 5 categories.

Assertions verify agent behavior, not output quality. They read from AgentTrace and check what the agent actually did.

Tool Call Assertions

AssertionWhat it checks
assert_tool_called(trace, "search")Tool was called at least once
assert_tool_not_called(trace, "delete")Tool was never called
assert_tool_call_order(trace, ["search", "email"])Tools called in exact order
assert_tool_call_count(trace, "search", expected_count=1)Called exactly N times
assert_no_tool_errors(trace)No tool calls produced errors
assert_tool_called_at_most(trace, "api", max_count=3)Called at most N times
assert_tool_allowlist(trace, ["search", "email"])Only these tools were called
assert_tool_denylist(trace, ["delete", "drop"])These tools were never called

State Assertions

AssertionWhat it checks
assert_state_consistent(trace)No state inconsistencies detected
assert_state_changed(trace, "user_count")State key was modified
assert_state_not_stale(trace, "cache", max_age_seconds=60)State is fresh
assert_state_consistent_across_traces(traces, "config")Same state across runs
assert_state_no_collisions(traces)No concurrent state conflicts

Governance Assertions

AssertionWhat it checks
assert_permission_respected(trace, "read_only")Agent respected permission boundary
assert_permission_violated(trace, "admin")Agent tried to exceed permissions
assert_tool_called_at_most(trace, "api", max_count=10)Rate limit respected
assert_approval_before_action(trace, "deploy", "confirm")Got approval before action

Resilience Assertions

AssertionWhat it checks
assert_graceful_degradation(trace, error_count)Agent handled errors gracefully
assert_no_silent_failure(trace, validators)No failures were silently ignored

Performance Assertions

AssertionWhat it checks
assert_latency(trace, max_ms=5000)Total execution time within limit
assert_token_usage(trace, max_tokens=10000)Token usage within limit
assert_step_count(trace, min_steps=2, max_steps=10)Agent took reasonable steps
assert_tool_latency(trace, "search", max_ms=1000)Specific tool latency within limit

Example

from sentinel.assertions import (
    assert_tool_called,
    assert_tool_call_order,
    assert_no_tool_errors,
    assert_graceful_degradation,
)

# After running your agent through the mock environment:
assert_tool_called(trace, "search")
assert_tool_called(trace, "email")
assert_tool_call_order(trace, ["search", "email"])
assert_no_tool_errors(trace)

# With chaos injection:
assert_graceful_degradation(trace, error_count=2)
💡 Tip Combine assertions with chaos injectors to test resilience. If an agent crashes when a tool fails, assert_no_tool_errors catches it.
