Custom Adapters

Write adapters for any agent framework using the hook-based protocol.

The Adapter Protocol

Sentinel's generic adapter (HookAdapter) works with any agent framework — no framework dependencies required. It follows a simple protocol:

  1. Accept a MockTool — provides canned responses instead of real execution.
  2. Accept an AgentTrace — records every call for behavioral assertions.
  3. Expose invoke() / __call__() — the tool invocation interface your agent calls.
  4. Record calls — every invocation is captured as a ToolCall in the trace with arguments, result, duration, and errors.

Framework-specific adapters (LangChain, CrewAI, OpenAI SDK) add an extra layer that matches each framework's tool protocol, but the core is always the same: mock the tool, record the call.

HookAdapter

The framework-agnostic adapter. HookAdapter wraps any callable with sentinel tracing:

from sentinel.adapters.generic import HookAdapter
from sentinel.env import MockTool
from sentinel.models import AgentTrace

trace = AgentTrace()

# Create a hook adapter with a mock tool
adapter = HookAdapter(
    mock=MockTool("search", response={"results": ["doc1"]}),
    trace=trace,
    name="search",
    description="Search the knowledge base",
)

# Execute it
result = adapter.execute(query="refund policy")
print(f"Result: {result}")
print(f"Trace calls: {trace.total_tool_calls}")

# Or call directly
result = adapter(query="return policy")

Parameters

  • mock — (Optional) A MockTool for canned responses. If None, requires fn.
  • trace — An AgentTrace to record all calls into.
  • name — (Optional) Tool name. Defaults to mock.name or the function name.
  • description — (Optional) Tool description.
  • fn — (Optional) The original function to wrap. When mock is None, this function is called directly.
⚠️ One is required You must provide either mock or fn — otherwise the adapter raises ValueError.

add_before_hook / add_after_hook

Before-hooks and after-hooks let you transform arguments and results without modifying the tool or the mock:

adapter = HookAdapter(
    mock=MockTool("search", response={"results": ["doc1"]}),
    trace=trace,
    name="search",
)

# Before-hook: transform kwargs before execution
# Receives kwargs dict, returns (possibly modified) kwargs dict
adapter.add_before_hook(
    lambda kwargs: {**kwargs, "limit": 10}
)

# After-hook: transform result after execution
# Receives (result, kwargs), returns (possibly modified) result
adapter.add_after_hook(
    lambda result, kwargs: result[:5]  # truncate results
)

# Error-hook: notified on exception (side-effect only)
# Receives (exception, kwargs), returns None
adapter.add_error_hook(
    lambda exc, kwargs: print(f"Error: {exc}")
)

# Execute — hooks run in registration order
result = adapter.execute(query="test")

Hook Execution Order

  1. Before-hooks — each receives the output of the previous one. Chain: kwargs₁ → kwargs₂ → kwargs₃.
  2. Tool execution — the mock (or original function) runs with the final kwargs.
  3. After-hooks — each receives the output of the previous one. Chain: result₁ → result₂ → result₃.
  4. Error-hooks — called only on exception. Side-effect only (logging, metrics). Does not modify the exception.

The @wrap_callable Decorator

A convenient decorator that replaces a function with a HookAdapter:

from sentinel.adapters.generic import wrap_callable
from sentinel.env import MockTool
from sentinel.models import AgentTrace

trace = AgentTrace()
search_mock = MockTool("search", response={"results": ["doc1"]})

@wrap_callable(mock=search_mock, trace=trace)
def search(**kwargs):
    """Search the knowledge base."""
    return real_search_impl(**kwargs)

# search is now a HookAdapter — call it like a function
result = search(query="test")

# The original function is preserved
print(search._original_fn)  # <function search at 0x...>

# Configure hooks on the adapter
adapter = search  # It IS the adapter
adapter.add_before_hook(lambda kwargs: {**kwargs, "limit": 10})

HookAdapterGroup

Manage multiple adapters as a logical unit. Useful for wrapping many tools and applying hooks uniformly:

from sentinel.adapters.generic import HookAdapterGroup
from sentinel.env import MockTool
from sentinel.models import AgentTrace

trace = AgentTrace()
group = HookAdapterGroup(trace=trace)

# Add tools to the group
group.add(mock=MockTool("search", response={"results": []}))
group.add(mock=MockTool("email", response="sent"))
group.add(mock=MockTool("database", response={"rows": 0}))

# Apply hooks to ALL adapters at once
group.apply_before_hook_to_all(
    lambda kwargs: {**kwargs, "timeout": 30}
)

# Access individual adapters
search = group.get("search")
result = search.execute(query="test")

Integration Checklist

When building a custom adapter for a new framework:

Step What to do
1. Identify the tool protocol How does your framework invoke tools? (e.g., .invoke(), .run(), __call__())
2. Create the adapter class Accept a MockTool and AgentTrace. Implement the invocation method to delegate to the mock and record into the trace.
3. Match the tool signature Your adapter must accept the same arguments the real tool does. Use the framework's schema if available.
4. Wire into the agent Replace the agent's tools with your adapters before running.
5. Assert on behavior Use assert_tool_called(), assert_no_tool_errors(), and trace inspection to verify behavior.

How to Test Custom Adapters

Test your adapter against the same criteria as the built-in ones:

from sentinel.adapters.generic import HookAdapter
from sentinel.env import MockTool
from sentinel.models import AgentTrace
from sentinel.assertions import assert_tool_called, assert_no_tool_errors

def test_custom_adapter_basic():
    """Adapter delegates to mock and records the call."""
    trace = AgentTrace()
    adapter = HookAdapter(
        mock=MockTool("test_tool", response={"ok": True}),
        trace=trace,
    )
    result = adapter.execute(input="hello")
    assert result == {"ok": True}
    assert_tool_called(trace, "test_tool")
    assert_no_tool_errors(trace)

def test_custom_adapter_error():
    """Adapter records errors and raises."""
    trace = AgentTrace()
    adapter = HookAdapter(
        mock=MockTool("fail_tool", error="Boom"),
        trace=trace,
    )
    try:
        adapter.execute()
        assert False, "Should have raised"
    except Exception:
        pass
    assert_tool_called(trace, "fail_tool")

def test_custom_adapter_hooks():
    """Before/after hooks transform args and results."""
    trace = AgentTrace()
    adapter = HookAdapter(
        mock=MockTool("search", response=["a", "b", "c"]),
        trace=trace,
    )
    adapter.add_before_hook(lambda kwargs: {**kwargs, "limit": 2})
    adapter.add_after_hook(lambda result, kwargs: result[:2])

    result = adapter.execute()
    assert result == ["a", "b"]
📖 Reference implementations See the source code for the built-in adapters: src/sentinel/adapters/langchain.py, crewai.py, openai.py, and generic.py. They all follow the same pattern: accept mock + trace, delegate execution, record calls.
