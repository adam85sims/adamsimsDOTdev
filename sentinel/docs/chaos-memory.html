Memory Pressure

Deep dive on MemoryPressure — simulate context window exhaustion, OOM kills, and GC pauses.

Overview

AI agents operating in production face memory pressure when their context windows fill up. Unlike ContextDegradation (which models gradual erosion), MemoryPressure models sudden pressure events: OOM kills, context window hard limits, eviction storms, and garbage collection pauses. This is the injector for testing what happens when the system runs out of room.

📖 Import path from sentinel.chaos import MemoryPressure

MemoryPressure tracks token usage in real time. As you call simulate_token_usage() to add tokens, the injector monitors whether the context exceeds its limits and triggers evictions or OOM kills accordingly.

Constructor Parameters

MemoryPressure(
    max_context_tokens: int = 4096,     # Hard limit on context window
    pressure_threshold: float = 0.8,     # When pressure starts (0.0–1.0)
    eviction_strategy: str = "fifo",     # "fifo", "priority", or "random"
    gc_pause_ms: float = 0.0,           # Simulated GC pause in ms
    oom_probability: float = 0.0,        # OOM kill chance when over limit
    seed: Optional[int] = None,          # Random seed for determinism
)

Parameter Details

  • max_context_tokens — The hard ceiling. When usage exceeds this, eviction or OOM is triggered. Default 4096 models a small context window; set to 128000 for GPT-4 class models.
  • pressure_threshold — The percentage of max_context_tokens at which the under_pressure flag becomes True and GC pauses begin. Default 0.8 = 80%.
  • eviction_strategy — How tokens are removed when over limit:
    • "fifo" — Remove oldest 20% of context (default)
    • "priority" — Remove least important 30%
    • "random" — Remove random 15–25%
  • gc_pause_ms — Simulated garbage collection pause in milliseconds. Only fires when under_pressure is True. Set to 0 to disable.
  • oom_probability — When context exceeds the limit, this is the probability of an OOM kill (context reset to 0) instead of a graceful eviction. Default 0.0 = always evict.

Token Usage Tracking

Properties

Property Type Description
current_usage int Current context usage in tokens
usage_percentage float Usage as fraction (0.0–1.0)
under_pressure bool True when usage ≥ pressure_threshold 
pressure = MemoryPressure(
    max_context_tokens=4096,
    pressure_threshold=0.8,
)

print(pressure.current_usage)      # 0
print(pressure.usage_percentage)   # 0.0
print(pressure.under_pressure)     # False

# Simulate adding tokens
result = pressure.simulate_token_usage(2000)
print(pressure.current_usage)      # 2000
print(pressure.usage_percentage)   # 0.488 (2000/4096)
print(pressure.under_pressure)     # False (48.8% < 80%)

result = pressure.simulate_token_usage(1500)
print(pressure.current_usage)      # 3500
print(pressure.usage_percentage)   # 0.854 (3500/4096)
print(pressure.under_pressure)     # True (85.4% >= 80%)

The simulate_token_usage() Method

Call this method to simulate adding tokens to the context window. It returns None on success, or an error message if eviction or OOM occurred:

result = pressure.simulate_token_usage(tokens: int) -> Optional[str]
# Returns None if successful
# Returns "OOM kill: ..." if OOM killed the context
# Returns "Context eviction: ..." if tokens were evicted

Behavior Flow

  1. Adds tokens to current_usage
  2. If current_usage > max_context_tokens:
    • Roll against oom_probability — if triggered, reset usage to 0 and return OOM message
    • Otherwise, evict tokens per eviction_strategy and return eviction message
  3. If under_pressure and gc_pause_ms > 0, sleep for the GC pause duration

OOM Kill Simulation

When context exceeds the limit and the OOM roll succeeds, the context window is reset to 0 — simulating an out-of-memory kill that wipes the agent's state:

pressure = MemoryPressure(
    max_context_tokens=1000,
    pressure_threshold=0.8,
    oom_probability=0.5,  # 50% chance of OOM vs eviction
    seed=42,
)

# Fill context past the limit
pressure.simulate_token_usage(800)   # 800/1000 = 80%, under_pressure = True
result = pressure.simulate_token_usage(400)  # 1200/1000, over limit!

if result and "OOM" in result:
    print(f"OOM kill! Context wiped: {result}")
    print(f"Usage after OOM: {pressure.current_usage}")  # 0
else:
    print(f"Evicted instead: {result}")
    print(f"Usage after eviction: {pressure.current_usage}")  # Reduced
⚠️ OOM resets context to 0 After an OOM kill, current_usage drops to 0. The agent loses all context — it must rebuild from scratch. This is the most severe outcome and tests whether your agent can recover from catastrophic memory failure.

Garbage Collection Pressure

When gc_pause_ms > 0 and the context is under_pressure, each simulate_token_usage() call introduces a real sleep to simulate GC pauses. This models the performance degradation that occurs when a system is under memory pressure and spends cycles on garbage collection.

import time

pressure = MemoryPressure(
    max_context_tokens=4096,
    pressure_threshold=0.7,
    gc_pause_ms=200,  # 200ms GC pauses
    seed=42,
)

# Below threshold: no GC pauses
start = time.time()
pressure.simulate_token_usage(2000)  # 48.8%, no pause
elapsed = time.time() - start
print(f"No pressure: {elapsed:.3f}s")  # ~0.001s

# Above threshold: GC pauses
pressure.simulate_token_usage(1000)  # 73.2%, no pause yet
start = time.time()
pressure.simulate_token_usage(100)   # 75.6%, still below 70%
elapsed = time.time() - start
print(f"Just below threshold: {elapsed:.3f}s")

start = time.time()
pressure.simulate_token_usage(200)   # 80.5%, now under pressure
elapsed = time.time() - start
print(f"Under pressure: {elapsed:.3f}s")  # ~0.2s (GC pause)

Eviction Strategies

When context exceeds the limit and OOM doesn't trigger, tokens are evicted according to the configured strategy:

Strategy Eviction Rate Behavior
fifo 20% of usage Remove oldest messages first — preserves recent context
priority 30% of usage Remove least important messages — more aggressive but smarter
random 15–25% of usage Random selection — unpredictable, tests agent robustness 
# FIFO: deterministic, removes oldest 20%
pressure_fifo = MemoryPressure(
    max_context_tokens=1000,
    eviction_strategy="fifo",
)

# Priority: aggressive, removes 30%
pressure_priority = MemoryPressure(
    max_context_tokens=1000,
    eviction_strategy="priority",
)

# Random: variable, 15-25%
pressure_random = MemoryPressure(
    max_context_tokens=1000,
    eviction_strategy="random",
    seed=42,
)

Wrapping Tools

Use wrap() to install memory pressure simulation on a MockTool. When the context is over limit, the wrapped tool may raise an OOM error or report an eviction:

from sentinel.chaos import MemoryPressure
from sentinel.env import MockTool

pressure = MemoryPressure(
    max_context_tokens=4096,
    pressure_threshold=0.8,
    oom_probability=0.1,
    seed=42,
)

tool = MockTool("agent_memory", response="context_data")
wrapped = pressure.wrap(tool)

# Simulate context filling up
for i in range(20):
    pressure.simulate_token_usage(300)

    try:
        result = wrapped(query=f"step_{i}")
        print(f"Step {i}: {result}")
    except MockToolError as e:
        print(f"Step {i}: Memory pressure failure — {e}")

Complete Test Example

import pytest
from sentinel.chaos import MemoryPressure
from sentinel.env import MockTool
from sentinel.assertions import assert_no_silent_failure

def test_agent_handles_memory_exhaustion():
    """Agent should recover when context window fills up."""
    pressure = MemoryPressure(
        max_context_tokens=4096,
        pressure_threshold=0.7,
        eviction_strategy="fifo",
        gc_pause_ms=100,
        oom_probability=0.2,
        seed=42,
    )

    tool = MockTool("search", response="results")
    wrapped = pressure.wrap(tool)

    # Simulate a long conversation that fills context
    outputs = []
    for step in range(20):
        tokens = 250 + (step * 50)  # Increasing token usage per step
        pressure.simulate_token_usage(tokens)

        try:
            result = run_agent_step(tool=wrapped, step=step)
            outputs.append(result)
        except Exception as e:
            outputs.append(None)

    # Agent should have produced some output
    valid_outputs = [o for o in outputs if o is not None]
    assert len(valid_outputs) > 0, "Agent produced no output under memory pressure"

    # Check final state
    print(f"Final usage: {pressure.current_usage}/{pressure.max_context_tokens}")
    print(f"Usage %: {pressure.usage_percentage:.1%}")
    print(f"Under pressure: {pressure.under_pressure}")
    print(f"Injections: {pressure.injection_count}")

    # Inspect injection records
    for record in pressure.records:
        print(f"  {record.failure_type}: {record.message}")

Full API Reference

class MemoryPressure:
    """Simulates memory pressure and context window exhaustion."""

    # Constructor
    def __init__(
        self,
        max_context_tokens: int = 4096,
        pressure_threshold: float = 0.8,
        eviction_strategy: str = "fifo",
        gc_pause_ms: float = 0.0,
        oom_probability: float = 0.0,
        seed: Optional[int] = None,
    ) -> None: ...

    # Properties
    injection_count: int          # Total pressure events
    current_usage: int            # Current token usage
    usage_percentage: float       # Usage as fraction (0.0–1.0)
    under_pressure: bool          # True when usage >= threshold

    # Core method
    def simulate_token_usage(self, tokens: int) -> Optional[str]:
        """Add tokens to context. Returns None on success, error on OOM/eviction."""

    # Wrapping
    def wrap(self, tool: MockTool) -> ChaosToolWrapper:
        """Wrap a MockTool with memory pressure simulation."""

    # State management
    def reset(self) -> None:
        """Reset all counters and usage."""

    def make_validator(self) -> Callable[[Any], bool]:
        """Create a validator for assert_no_silent_failure."""
