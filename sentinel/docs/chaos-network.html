Network Partitions

Deep dive on NetworkPartition — simulate partial connectivity, service isolation, and healing.

Overview

In distributed microservice architectures, network partitions are a fact of life. Services lose connectivity to each other due to DNS failures, cloud provider outages, firewall misconfigurations, or infrastructure upgrades. NetworkPartition models these scenarios by defining a connectivity matrix that controls which services can reach which other services.

📖 Import path from sentinel.chaos import NetworkPartition

When a partition is active, calls between disconnected services fail with TimeoutError, simulating a connection that never completes.

The Connectivity Matrix

The connectivity parameter is a dictionary mapping each service name to the list of services it can reach. If service A is not in B's reachable list, any call from A to B will fail.

partition = NetworkPartition(
    connectivity={
        "api": ["database", "cache"],       # api can reach db and cache
        "database": ["api"],                  # db can reach api only
        "cache": ["api"],                     # cache can reach api only
        "external_api": [],                   # isolated — can't reach anything
        "webhook": [],                        # isolated — can't reach anything
    },
    partition_probability=1.0,  # Always partitioned
)
⚠️ Asymmetric partitions The connectivity matrix is not automatically symmetric. If A can reach B, B does not necessarily reach A. This models real-world asymmetric partitions (e.g., a firewall rule that blocks inbound but not outbound traffic).

Connectivity Examples

Scenario Connectivity Matrix
Full isolation {"db": [], "api": [], "cache": []}
API ↔ DB only {"api": ["db"], "db": ["api"], "cache": []}
External services down {"api": ["db", "cache"], "db": ["api"], "ext": []}
Star topology {"hub": ["a","b","c"], "a": ["hub"], "b": ["hub"], "c": ["hub"]}

Constructor Parameters

NetworkPartition(
    connectivity: Dict[str, List[str]],  # Service reachability matrix
    partition_probability: float = 1.0,  # 0.0–1.0, chance partition is active
    seed: Optional[int] = None,          # Random seed for determinism
    heal_after_calls: Optional[int] = None,  # Heal after N failed calls
)

Parameter Details

  • connectivity — The core definition. Maps each service name to the list of services it can reach. Services not in the dict are treated as unreachable from all sources.
  • partition_probability — At 1.0 (default), every call checks connectivity. At lower values, some calls pass through even to disconnected services (simulating intermittent connectivity).
  • seed — Deterministic random seed for the probability check.
  • heal_after_calls — When set, the partition automatically heals after this many failed calls. Useful for testing recovery behavior.

Key Methods

is_reachable(source, target)

Check whether source can reach target given the current connectivity matrix and partition state:

partition = NetworkPartition(
    connectivity={
        "api": ["database", "cache"],
        "database": ["api"],
        "cache": [],
    },
)

print(partition.is_reachable("api", "database"))   # True
print(partition.is_reachable("api", "external"))    # False
print(partition.is_reachable("cache", "api"))       # False (cache can't reach api)
print(partition.is_reachable("database", "api"))    # True

partition_service() / restore_service()

After creating a partition, you can dynamically modify which services are isolated:

partition = NetworkPartition(
    connectivity={
        "api": ["database", "cache", "search"],
        "database": ["api"],
        "cache": ["api"],
        "search": ["api"],
    },
)

# Initially, api can reach search
assert partition.is_reachable("api", "search")  # True

# Dynamically isolate search
partition.partition_service("search")  # Makes search unreachable
assert not partition.is_reachable("api", "search")  # False

# Restore connectivity
partition.restore_service("search")
assert partition.is_reachable("api", "search")  # True

check_connectivity()

Returns the current effective connectivity state — useful for logging and debugging:

state = partition.check_connectivity()
# Returns dict with current connectivity matrix and active status

get_isolated_services()

Returns a list of services that are currently unable to reach any other service:

partition = NetworkPartition(
    connectivity={
        "api": ["database"],
        "database": ["api"],
        "cache": [],       # isolated
        "search": [],      # isolated
    },
)

isolated = partition.get_isolated_services()
print(isolated)  # ["cache", "search"]

Wrapping Tools

Use wrap() to install network partition simulation on a MockTool. When the tool is called and the target service is unreachable, a TimeoutError is raised:

from sentinel.chaos import NetworkPartition
from sentinel.env import MockTool

# Set up tools
api_tool = MockTool("api", response={"status": 200})
cache_tool = MockTool("cache", response={"data": "cached"})

# Define partition: api can reach database but not cache
partition = NetworkPartition(
    connectivity={
        "api": ["database"],
        "database": ["api"],
        "cache": [],
    },
    partition_probability=1.0,
)

# Wrap tools — calls to disconnected services fail
wrapped_api = partition.wrap(api_tool)
wrapped_cache = partition.wrap(cache_tool)

# Api tool works (it can reach its target)
result = wrapped_api(endpoint="/users")  # OK

# Cache tool fails (it's isolated)
try:
    result = wrapped_cache(key="user:123")
except TimeoutError as e:
    print(f"Network partition: {e}")

Partial Degradation

Beyond full partitions, you can model partial degradation with latency, packet loss, and jitter. This is common in cross-region deployments or during cloud provider incidents.

Latency

# Add artificial latency to tool calls
from sentinel.chaos import NetworkPartition

partition = NetworkPartition(
    connectivity={
        "api": ["database", "cache"],
        "database": ["api"],
    },
    partition_probability=0.3,  # 30% chance of partition per call
    seed=42,
)

# Combine with tool wrappers that add latency
# The partition handles connectivity; add latency separately:
import time

original_tool = MockTool("api", response="ok")
wrapped = partition.wrap(original_tool)

Packet Loss & Jitter

While the base NetworkPartition handles connectivity, you can combine it with custom tool wrappers to add packet loss and jitter effects:

# Simulate 20% packet loss + 100ms jitter on API calls
class DegradedApiTool:
    def __init__(self, tool, loss_rate=0.2, jitter_ms=100):
        self._tool = tool
        self._loss_rate = loss_rate
        self._jitter_ms = jitter_ms
        self._rng = random.Random(seed)

    def __call__(self, **kwargs):
        # Simulate packet loss
        if self._rng.random() < self._loss_rate:
            raise TimeoutError("Packet lost — connection reset")

        # Simulate jitter
        jitter = self._rng.uniform(0, self._jitter_ms / 1000)
        time.sleep(jitter)

        return self._tool(**kwargs)

Auto-Healing with heal_after_calls

The heal_after_calls parameter lets you test recovery behavior. After the specified number of failed calls, the partition heals and all services become reachable again:

partition = NetworkPartition(
    connectivity={
        "api": ["database"],
        "database": ["api"],
        "cache": [],
    },
    partition_probability=1.0,
    heal_after_calls=3,  # Heal after 3 failed calls
)

tool = MockTool("cache", response="data")
wrapped = partition.wrap(tool)

# Calls 1–3: fail (partition active)
for i in range(3):
    try:
        wrapped(key=f"item:{i}")
    except TimeoutError:
        print(f"Call {i+1}: partition active")

# Call 4+: succeeds (partition healed)
result = wrapped(key="item:4")
print(f"Call 4: succeeded with {result}")
💡 Reset for new test runs Call partition.reset() to restore the partition to its initial state after a test run. This is essential for test isolation.

Real-World Scenarios

Cloud Provider Outage

Simulate a scenario where one availability zone goes down, isolating a subset of services:

cloud_outage = NetworkPartition(
    connectivity={
        "api_az1": ["database", "cache", "queue"],
        "api_az2": ["database", "cache", "queue"],
        "database": ["api_az1", "api_az2"],       # Cross-AZ replication
        "cache_az1": ["api_az1"],                   # Only local cache
        "cache_az2": ["api_az2"],
        "queue_az2": [],                            # Queue service is down
    },
    partition_probability=1.0,
    heal_after_calls=10,  # Recovery after 10 failures
)

DNS Failure

Simulate DNS resolution failures where external services are unreachable:

dns_failure = NetworkPartition(
    connectivity={
        "agent": ["local_tools", "cache"],
        "local_tools": ["agent", "database"],
        "external_api": [],     # DNS resolution fails
        "search_engine": [],    # DNS resolution fails
        "webhook": [],          # DNS resolution fails
    },
    partition_probability=0.7,  # 70% of external calls fail
    seed=42,
)

Split-Brain Network

Simulate a split-brain scenario where two halves of the system can't communicate:

split_brain = NetworkPartition(
    connectivity={
        "leader": ["replica1", "replica2"],
        "replica1": ["leader"],
        "replica2": ["leader"],
        # Left half
        "service_a": ["service_b"],
        "service_b": ["service_a"],
        # Right half
        "service_c": ["service_d"],
        "service_d": ["service_c"],
        # Cross-half: isolated!
        "service_a": [],  # Can't reach right half
        "service_c": [],  # Can't reach left half
    },
    partition_probability=1.0,
)

Complete Test Example

import pytest
from sentinel.chaos import NetworkPartition
from sentinel.env import MockTool
from sentinel.assertions import assert_no_silent_failure

def test_agent_handles_network_partition():
    """Agent should handle partial service isolation gracefully."""
    # Set up tools
    api_tool = MockTool("api", response={"status": 200})
    cache_tool = MockTool("cache", response={"data": "cached"})

    # Define partition
    partition = NetworkPartition(
        connectivity={
            "api": ["database"],
            "cache": [],  # Cache is isolated
        },
        partition_probability=1.0,
        heal_after_calls=5,
        seed=42,
    )

    # Wrap tools
    wrapped_api = partition.wrap(api_tool)
    wrapped_cache = partition.wrap(cache_tool)

    # Run agent — it should handle cache failures
    output = run_agent(api=wrapped_api, cache=wrapped_cache)

    # Assertions
    assert output is not None
    assert_no_silent_failure(output)

    # Verify partition occurred
    assert partition.injection_count > 0

    # Verify agent adapted (used fallback, retried, etc.)
    isolated = partition.get_isolated_services()
    assert "cache" in isolated

Full API Reference

class NetworkPartition:
    """Simulates network partitions between services."""

    # Constructor
    def __init__(
        self,
        connectivity: Dict[str, List[str]],
        partition_probability: float = 1.0,
        seed: Optional[int] = None,
        heal_after_calls: Optional[int] = None,
    ) -> None: ...

    # Properties
    injection_count: int  # Total partition failures injected

    # Connectivity checks
    def is_reachable(self, source: str, target: str) -> bool:
        """Check if source can reach target."""

    def check_connectivity(self) -> Dict:
        """Get current connectivity state."""

    def get_isolated_services(self) -> List[str]:
        """Get services that can't reach anything."""

    # Dynamic partition management
    def partition_service(self, service: str) -> None:
        """Isolate a service from all others."""

    def restore_service(self, service: str) -> None:
        """Restore a service's connectivity."""

    # Wrapping
    def wrap(self, tool: MockTool) -> ChaosToolWrapper:
        """Wrap a MockTool with network partition simulation."""

    # State management
    def reset(self) -> None:
        """Reset partition state."""

    def make_validator(self) -> Callable[[Any], bool]:
        """Create a validator for assert_no_silent_failure."""
