sentinel.env — Mock Environment Layer

Provides configurable mock tools, REST/GraphQL API simulation, in-memory database, and a fluent builder for composing test environments.

Imports: from sentinel.env import MockTool, MockAPI, MockDatabase, EnvironmentBuilder, Environment

Errors

class MockToolError(Exception):
__init__(message: str, status_code: int = 500)

Base error for mock tool failures. Raised when a mock tool simulates an error response.

class RateLimitError(MockToolError):
__init__(message: str = "Rate limit exceeded", retry_after: float = 60.0)

Raised when a mock tool simulates rate limiting (HTTP 429).

class TimeoutError(MockToolError):
__init__(message: str = "Request timed out")

Raised when a mock tool simulates a timeout (HTTP 408).

sentinel.assertions — Behavioral Assertions

20+ assertions across tool calls, state consistency, governance compliance, resilience, and performance. Each assertion takes an AgentTrace and raises AssertionError with a descriptive message on failure.

Imports: from sentinel.assertions import assert_tool_called, assert_no_tool_errors, ...

Tool Call Assertions

assert_tool_called(trace: AgentTrace, tool_name: str, **expected_kwargs)

Assert that a specific tool was called. If kwargs are provided, checks that at least one call matches ALL specified arguments.

assert_tool_not_called(trace: AgentTrace, tool_name: str)

Assert that a specific tool was NOT called. Useful for verifying that agents don't take unauthorized actions.

assert_tool_call_order(trace: AgentTrace, expected_order: List[str])

Assert that tools were called in the expected order. Checks that UNIQUE tool names appear in the trace in the specified order. Extra calls between expected calls are allowed.

assert_tool_call_count(trace: AgentTrace, tool_name: str, expected_count: int)

Assert that a tool was called exactly N times.

assert_no_tool_errors(trace: AgentTrace)

Assert that no tool calls resulted in errors. Reports all failed calls on failure.

State Assertions

assert_state_consistent(trace: AgentTrace, key: str, expected: Any = None)

Assert that agent state has a specific value. Checks the most recent state change for the given key.

assert_state_changed(trace: AgentTrace, key: str)

Assert that a specific state key was modified during execution.

assert_state_not_stale(trace: AgentTrace, key: str, max_age_seconds: float)

Assert that a state key was recently written to. Detects stale state where the agent reads outdated data.

assert_state_consistent_across_traces(traces: List[AgentTrace], key: str)

Assert that a state key has the same value across multiple traces. Detects cross-session divergence.

assert_state_no_collisions(traces: List[AgentTrace], allowed_keys: Optional[List[str]] = None)

Assert that no state key collisions occurred between agents. Detects concurrent writes to the same key.

Governance Assertions

assert_permission_respected(trace: AgentTrace, tool_name: str, requires_approval: bool = True)

Assert that a tool requiring approval was called (and thus approved).

assert_permission_violated(trace: AgentTrace, tool_name: str)

Assert that a forbidden tool WAS called (detecting a violation). Asserts the NEGATIVE — that the agent took an action it should not have.

assert_tool_called_at_most(trace: AgentTrace, tool_name: str, max_count: int)

Assert that a tool was called at most N times. Implements rate-limited governance for expensive or dangerous tools.

assert_tool_allowlist(trace: AgentTrace, allowed_tools: List[str])

Assert that ONLY tools from the allowlist were called. The agent may only invoke tools in the allowed set.

assert_tool_denylist(trace: AgentTrace, denied_tools: List[str])

Assert that NONE of the denied tools were called. Certain dangerous tools must never be invoked under any circumstances.

assert_approval_before_action(trace: AgentTrace, approval_tool: str, action_tool: str)

Assert that an approval tool was called before a restricted action. Implements sequential governance: certain actions require an approval step first.

Resilience Assertions

assert_graceful_degradation(trace: AgentTrace, on_error_tool: Optional[str] = None)

Assert that the agent handled errors gracefully. Checks that the trace completed, no critical errors are unrecovered, and if a specific tool errored the agent continued execution.

assert_no_silent_failure(trace: AgentTrace, validator: Optional[Callable] = None)

Assert that the agent's output is not a silent failure. Detects when the agent completes without errors but produces incorrect or empty output.

Performance Assertions

assert_latency(trace: AgentTrace, max_ms: Optional[float] = None,
min_ms: Optional[float] = None, per_step_max_ms: Optional[float] = None)

Assert that execution latency is within bounds. Checks total time and optionally per-step timing. Detects performance regressions.

assert_token_usage(trace: AgentTrace, max_tokens: Optional[int] = None,
min_tokens: Optional[int] = None)

Assert that token usage is within bounds. Reads from trace.metadata['token_usage']['total_tokens'].

assert_step_count(trace: AgentTrace, max_steps: Optional[int] = None,
min_steps: Optional[int] = None, exact_steps: Optional[int] = None)

Assert that the agent used an expected number of steps. Detects agents stuck in loops or doing insufficient work.

assert_tool_latency(trace: AgentTrace, tool_name: str,
max_ms: Optional[float] = None, avg_max_ms: Optional[float] = None)

Assert latency bounds on a specific tool's calls. Detects API degradation or agents hammering slow tools.

Helper Function

detect_state_collisions(traces: List[AgentTrace]) → List[Dict]

Detect when multiple agents write to the same state key within a 1-second window. Returns a list of collision events. Use assert_state_no_collisions() for assertion-based testing.

ScenarioRunner

class ScenarioRunner:
run(scenario: SentinelScenario, agent_fn: Optional[Callable] = None,
env: Optional[Environment] = None) → SentinelResult
run_batch(scenarios: List[SentinelScenario],
agent_fn: Optional[Callable] = None) → List[SentinelResult]

Executes test scenarios against agents. The runner: (1) builds the environment from config, (2) constructs the agent, (3) creates an AgentTrace, (4) invokes the agent's task, (5) runs assertions against the trace, (6) returns a SentinelResult.

The agent_fn callable receives (task, env, trace) and should run the agent, recording behavior into the trace.

Pytest decorator for declarative sentinel test definition. Wraps a test function so it receives a pre-built trace (AgentTrace) and env (Environment) as keyword arguments. Handles environment building, chaos wiring, and timing.

still_failScenario fails in both runs
new_scenarioScenario exists only in current run
removedScenario exists only in baseline
