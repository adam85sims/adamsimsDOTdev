Context Degradation

Test how agents handle context window erosion via truncation, noise, and drift.

Overview

Long-running agents inevitably face context window erosion. Earlier instructions get dropped as new messages fill the window, tool results accumulate noise, and the system prompt drifts from its original intent. ContextDegradation models these three failure modes deterministically, letting you test whether your agent maintains correctness as its context degrades.

📖 Import path from sentinel.chaos import ContextDegradation, DegradationStrategy

Unlike ToolFailureInjector (which intercepts individual calls), ContextDegradation operates on the step level. You call on_step() each time the agent advances, passing in the current context and system prompt. The injector returns degraded versions.

Three Degradation Strategies

1. Truncation

Progressively removes earlier context entries as steps accumulate. This simulates what happens when a context window fills up — older messages are dropped to make room for new ones. The system prompt (first entry, if present) is always preserved.

  • At low degradation: 1–2 early entries dropped
  • At high degradation: up to max_truncation_pct of entries dropped from the front
  • System prompt is never truncated

2. Noise

Injects random character substitutions into tool result content. This simulates signal degradation — garbled API responses, corrupted data from partial transfers, or encoding issues in long-running sessions.

  • Noise rate scales with degradation level
  • At level 0.1: ~1% of characters perturbed
  • At level 1.0: up to 10% of characters corrupted
  • Non-string content is left unchanged

3. Drift

Gradually shifts system prompt instructions via character-level perturbations (swapping adjacent characters in words). This simulates instruction drift — where the agent's understanding of its task slowly diverges from the original spec.

  • Drift accumulates across steps (compound effect)
  • Short words (≤2 chars) are never drifted
  • At high levels, critical instructions may become garbled

Constructor Parameters

ContextDegradation(
    strategy: str = "truncation",   # "truncation", "noise", or "drift"
    start_step: int = 1,            # Step at which degradation begins
    degradation_rate: float = 0.1,  # Speed of degradation (0.0–1.0)
    max_truncation_pct: float = 0.5,# Max fraction to truncate (0.0–1.0)
    seed: Optional[int] = None,     # Random seed for determinism
)

Parameter Details

  • strategy — Maps to the DegradationStrategy enum: "truncation", "noise", or "drift".
  • start_step — Steps before this number run with zero degradation. Default 1 means degradation starts immediately.
  • degradation_rate — Controls the acceleration curve. 0.1 = mild erosion over many steps. 0.5 = aggressive. 1.0 = near-instant maximum degradation.
  • max_truncation_pct — For the truncation strategy: the maximum fraction of context that can be dropped. Default 0.5 means at most 50% of entries are removed.
  • seed — Ensures deterministic noise/drift injection across runs.

Quadratic Acceleration Curve

Context degradation does not increase linearly. The current_degradation_level property uses a quadratic acceleration curve that models real context window behavior — the last 20% of the window is much worse than the first 20%:

@property
def current_degradation_level(self) -> float:
    if self._step_count <= self.start_step:
        return 0.0
    elapsed = self._step_count - self.start_step
    raw = elapsed * self.degradation_rate
    return min(1.0, raw * raw)  # Quadratic!

💡 Why quadratic? Real context windows exhibit accelerating degradation. A system at 90% context utilization behaves far worse than one at 50% — the remaining space can't absorb new information effectively. The quadratic curve models this compounding effect.

Truncation Example

from sentinel.chaos import ContextDegradation

degradation = ContextDegradation(
    strategy="truncation",
    start_step=3,
    degradation_rate=0.2,
    max_truncation_pct=0.4,
)

# Build a context with system prompt + messages
context = [
    {"role": "system", "content": "You are a helpful research assistant."},
    {"role": "user", "content": "Find information about quantum computing."},
    {"role": "assistant", "content": "I'll search for that."},
    {"role": "tool", "content": "Found 5 papers on quantum computing."},
    {"role": "user", "content": "Summarize the first paper."},
]

# Step 1–2: no degradation
result = degradation.on_step(step_id=1, context=context)
print(result["injected"])  # False

result = degradation.on_step(step_id=2, context=context)
print(result["injected"])  # False

# Step 3+: degradation begins
result = degradation.on_step(step_id=3, context=context)
print(result["injected"])   # True
print(result["degradation_level"])  # 0.01
print(f"Context entries: {len(result['context'])}")  # May be truncated

# Later steps: more truncation
for step in range(4, 10):
    result = degradation.on_step(step_id=step, context=context)
    if result["injected"]:
        print(f"Step {step}: level={result['degradation_level']:.2f}, "
              f"entries={len(result['context'])}")

# Noise degrades tool results — garbled characters appear

# e.g., "Q!ery r3turned 1,247 r3cords with a4v t3mperature 72.3°F"

degradation = ContextDegradation(

The on_step() Method

Call on_step() each time your agent advances a step. It returns a dict with the degraded context/prompt and metadata:

result = degradation.on_step(
    step_id: int,                          # Current step number
    context: Optional[List[Dict]] = None,  # Context entries (for truncation/noise)
    system_prompt: Optional[str] = None,   # System prompt (for drift)
)

Return Value

Key Type Description
context List[Dict] Possibly truncated/noised context list
system_prompt str Possibly drifted system prompt
degradation_level float Current intensity (0.0 = none, 1.0 = max)
injected bool Whether degradation was applied this step
⚠️ Pass the right inputs Truncation and noise strategies require the context parameter. The drift strategy requires the system_prompt parameter. If you pass the wrong one, no degradation is applied (the injector silently skips).

The make_validator() Pattern

Use make_validator() to create an assertion-compatible function that checks whether the agent's output is valid given the degradation applied:

from sentinel.chaos import ContextDegradation
from sentinel.assertions import assert_no_silent_failure

degradation = ContextDegradation(
    strategy="truncation",
    degradation_rate=0.3,
)

# Run agent steps
context = [...]
for step_id in range(1, 20):
    result = degradation.on_step(step_id=step_id, context=context)
    context = result["context"]  # Use degraded context

# Create validator
validator = degradation.make_validator(
    original_context=original_context,   # Pre-degradation reference
    original_prompt=original_prompt,     # Pre-degradation reference
)

# Assert output validity
output = run_agent(context=context)
assert_no_silent_failure(output, validator=validator)

Validator Behavior by Degradation Level

Level Validation Rule
< 0.3 Any non-None output is valid (mild degradation)
0.3 – 1.0 Output must be a non-empty string
N/A (None) Output is invalid

Complete Example

import pytest
from sentinel.chaos import ContextDegradation, DegradationStrategy
from sentinel.env import MockTool

def test_agent_survives_truncation():
    """Agent should still produce valid output as context degrades."""
    degradation = ContextDegradation(
        strategy="truncation",
        start_step=3,
        degradation_rate=0.2,
        max_truncation_pct=0.4,
        seed=42,
    )

    context = [
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "What is the capital of France?"},
    ]

    # Run 10 agent steps with degrading context
    for step_id in range(1, 11):
        result = degradation.on_step(step_id=step_id, context=context)
        context = result["context"]
        assert result["degradation_level"] >= 0

    # After degradation, agent should still produce something
    output = run_agent(context=context)
    assert output is not None
    assert len(str(output).strip()) > 0

    # Inspect records
    assert degradation.injection_count > 0
    for record in degradation.records:
        print(f"Step {record.step_id}: {record.failure_type} - {record.message}")

Full API Reference

class ContextDegradation:
    """Simulates context window erosion during long-running tasks."""

    # Constructor
    def __init__(
        self,
        strategy: str = "truncation",
        start_step: int = 1,
        degradation_rate: float = 0.1,
        seed: Optional[int] = None,
        max_truncation_pct: float = 0.5,
    ) -> None: ...

    # Properties
    injection_count: int              # Number of degradation applications
    records: List[InjectionRecord]    # Injection history
    current_degradation_level: float  # 0.0–1.0 (quadratic acceleration)

    # Core method
    def on_step(
        self,
        step_id: int,
        context: Optional[List[Dict[str, Any]]] = None,
        system_prompt: Optional[str] = None,
    ) -> Dict[str, Any]:
        """Process a step and return degraded context/prompt."""

    # Validator
    def make_validator(
        self,
        original_context: Optional[List[Dict[str, Any]]] = None,
        original_prompt: Optional[str] = None,
    ) -> Callable[[Any], bool]:
        """Create a validator for assert_no_silent_failure."""

    def reset(self) -> None:
        """Reset degradation state."""
