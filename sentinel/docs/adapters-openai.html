OpenAI SDK Adapter

Test OpenAI Agents SDK with Sentinel's mock tools, traces, and chaos injection.

Installation

Install the OpenAI SDK adapter as an optional extra:

pip install sentinel[openai]

This pulls in openai-agents (≥ 0.1) for FunctionTool, function_tool, and RunContextWrapper. Without the extra, the adapter duck-types the FunctionTool interface so it works for standalone mock-only testing.

wrap_openai_agent()

The primary entry point. wrap_openai_agent() wraps an OpenAI Agents SDK Agent with sentinel tool mocking, creating an OpenAIAgentWrapper with adapters ready for wiring:

from sentinel.adapters.openai import wrap_openai_agent
from sentinel.env import MockTool
from sentinel.models import AgentTrace

trace = AgentTrace()

# Define mock responses for every tool
tool_map = {
    "search": MockTool("search", response={"results": ["doc1"]}),
    "email": MockTool("email", response="sent"),
    "database": MockTool("database", response={"rows": 42}),
}

# Wrap the agent
wrapper = wrap_openai_agent(
    agent=my_openai_agent,
    tool_map=tool_map,
    trace=trace,
)

# Wire mock tools into the agent's tools list
my_openai_agent.tools = wrapper.get_tool_list()

Parameters

  • agent — An OpenAI Agents SDK Agent instance.
  • tool_map — Dict[str, MockTool] mapping tool names to mock implementations.
  • trace — An AgentTrace to record all calls into.

SentinelFunctionTool

The adapter class that wraps a MockTool as an OpenAI Agents SDK function tool. Each entry in the tool_map becomes one of these.

from sentinel.adapters.openai import SentinelFunctionTool
from sentinel.env import MockTool
from sentinel.models import AgentTrace

trace = AgentTrace()

# Create a function tool from a mock
tool = SentinelFunctionTool(
    mock=MockTool("search", response=SEARCH_RESULTS),
    trace=trace,
    name="search",
    description="Search the knowledge base",
)

# Invoke with a JSON string (OpenAI SDK convention)
result = tool.invoke('{"query": "refund policy"}')

# Or invoke with a dict
result = tool.invoke({"query": "refund policy"})

# Or call directly
result = tool(query="refund policy")

Key Features

  • JSON schema support — params_json_schema property provides a JSON schema for the tool's input parameters, matching the SDK's FunctionTool interface.
  • Async support — async_invoke() for async runners (delegates to the sync mock internally).
  • Flexible input — Accepts JSON strings, dicts, or None.

Tool Map Usage

The tool_map is a Dict[str, MockTool]. Each key must match the tool name your OpenAI agent expects:

tool_map = {
    "search": MockTool("search", response={"results": ["doc1"]}),
    "email": MockTool("email", response="sent"),
}

# The wrapper creates SentinelFunctionTool for each entry
wrapper = wrap_openai_agent(
    agent=my_agent,
    tool_map=tool_map,
    trace=trace,
)

# Access individual adapters
search_adapter = wrapper.get_adapter("search")
print(f"Search calls: {search_adapter.mock.call_count}")

# Get all tools as a list for wiring into the agent
my_agent.tools = wrapper.get_tool_list()

Full Example with OpenAI Agents SDK

Here's a complete example integrating with the OpenAI Agents SDK:

from agents import Agent, Runner
from sentinel.adapters.openai import wrap_openai_agent
from sentinel.chaos import inject_failures
from sentinel.env import MockTool
from sentinel.models import AgentTrace
from sentinel.assertions import assert_tool_called, assert_no_tool_errors

trace = AgentTrace()

# 1. Define mock tools
search_mock = MockTool("search", response={"results": ["Sentinel docs"]})

# 2. Optionally inject chaos
inject_failures(search_mock, failure_rate=0.2, error_type="TimeoutError")

# 3. Create the OpenAI agent
agent = Agent(
    name="Research Assistant",
    instructions="Search for information and summarize findings.",
    tools=[],  # Will be replaced by mock tools
)

# 4. Wrap with sentinel
wrapper = wrap_openai_agent(
    agent=agent,
    tool_map={"search": search_mock},
    trace=trace,
)

# 5. Wire in the mock tools
agent.tools = wrapper.get_tool_list()

# 6. Run the agent
result = Runner.run_sync(agent, "Find information about Sentinel testing")

# 7. Assert on behavior
assert_tool_called(trace, "search")
assert_no_tool_errors(trace)

print(f"Agent output: {result.final_output}")
print(f"Trace calls: {trace.total_tool_calls}")
✅ Key Insight The adapter replaces only the tool execution layer — the agent's LLM reasoning loop, prompt handling, and output formatting all remain untouched. This means your tests exercise the agent's actual decision-making logic.
