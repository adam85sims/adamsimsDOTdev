CrewAI Adapter

Test CrewAI crews with Sentinel's mock tools, traces, and chaos injection.

Installation

Install the CrewAI adapter as an optional extra:

pip install sentinel[crewai]

This pulls in crewai (≥ 0.80) for BaseTool and the @tool decorator. Without the extra, the adapter duck-types enough of CrewAI's interface to work for standalone mock-only testing.

SentinelCrewTool

The core building block. SentinelCrewTool wraps a Sentinel MockTool as a CrewAI-compatible tool that exposes run(**kwargs) — CrewAI's BaseTool invocation protocol.

from sentinel.adapters.crewai import SentinelCrewTool
from sentinel.env import MockTool
from sentinel.models import AgentTrace

trace = AgentTrace()

# Create a CrewAI-compatible tool from a mock
adapter = SentinelCrewTool(
    mock=MockTool("search", response={"results": ["doc1", "doc2"]}),
    trace=trace,
    name="search",
    description="Search the knowledge base",
)

# Use it like a CrewAI BaseTool — call .run()
result = adapter.run(query="refund policy")
print(f"Mock response: {result}")

# Also works with _run() (some CrewAI patterns call this directly)
result = adapter._run(query="return policy")

Parameters

  • mock — A sentinel.env.MockTool providing canned responses and error injection.
  • trace — An AgentTrace that records every call with arguments, results, duration, and errors.
  • name — (Optional) Tool name. Defaults to mock.name.
  • description — (Optional) Tool description shown to the LLM. Defaults to "Sentinel mock tool: {name}".

Wrapping an Entire Crew

Use wrap_crew_agent() to wrap a CrewAI Crew at once. This creates a CrewAgentWrapper with adapters ready for wiring into your agents:

from sentinel.adapters.crewai import wrap_crew_agent
from sentinel.env import MockTool
from sentinel.models import AgentTrace

trace = AgentTrace()

# Define mock responses for every tool
tool_map = {
    "search": MockTool("search", response={"results": ["doc1"]}),
    "email": MockTool("email", response="sent"),
    "database": MockTool("database", response={"rows": 42}),
}

# Wrap the crew
wrapper = wrap_crew_agent(
    crew=my_crew,
    tool_map=tool_map,
    trace=trace,
)

# Wire the mock tools into each agent's tool list
for agent in my_crew.agents:
    agent.tools = wrapper.get_tool_list()

# Run the crew as usual — tools are intercepted
result = my_crew.kickoff()

# Inspect what happened
for name, adapter in wrapper.adapters.items():
    print(f"{name}: called {adapter.mock.call_count} times")
print(f"Total trace calls: {trace.total_tool_calls}")

Tool Map

The tool_map is a Dict[str, MockTool] mapping tool names to their mock implementations. Every key must match the tool name your CrewAI agents expect. The wrapper creates a SentinelCrewTool for each entry.

Accessing Individual Adapters

The CrewAgentWrapper provides several ways to inspect individual tool adapters:

# Get a specific adapter by name
search_adapter = wrapper.get_adapter("search")
print(f"Search calls: {search_adapter.mock.call_count}")

# Get the underlying MockTool for a name
search_mock = wrapper.get_mock("search")
print(f"Search responses: {search_mock.responses}")

# Get all adapters as a dict
all_adapters = wrapper.adapters  # Dict[str, SentinelCrewTool]

# Get all adapters as a list (for passing to Agent.tools directly)
tool_list = wrapper.get_tool_list()  # List[SentinelCrewTool]

Example with Chaos Injection

Combine the CrewAI adapter with Sentinel's chaos layer to test crew resilience under failure:

from sentinel.adapters.crewai import wrap_crew_agent
from sentinel.chaos import inject_failures
from sentinel.env import MockTool
from sentinel.models import AgentTrace

trace = AgentTrace()

# Create a mock tool, then inject chaos
search_mock = MockTool("search", response={"results": ["doc1"]})
inject_failures(search_mock, failure_rate=0.3, error_type="TimeoutError")

email_mock = MockTool("email", response="sent")
inject_failures(email_mock, failure_rate=0.1, error_type="ConnectionError")

# Wrap crew with chaotic mocks
wrapper = wrap_crew_agent(
    crew=my_crew,
    tool_map={"search": search_mock, "email": email_mock},
    trace=trace,
)

# Wire into agents
for agent in my_crew.agents:
    agent.tools = wrapper.get_tool_list()

# Run the crew — agents must handle tool failures gracefully
result = my_crew.kickoff()

# Analyze results
errors = [tc for tc in trace.tool_calls if tc.error]
print(f"Total calls: {trace.total_tool_calls}")
print(f"Errors: {len(errors)}")
for err in errors:
    print(f"  {err.tool_name}: {err.error}")
📖 Learn more See Tool Failures for all failure injection options (latency, error types, partial responses) and Agent Traces for inspecting recorded calls.
