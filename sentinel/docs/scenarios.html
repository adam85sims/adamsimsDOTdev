YAML Scenarios

Define test scenarios declaratively in YAML files.

Overview

YAML Scenarios let you define behavioral tests without writing Python. A scenario file specifies the task, mock tools, chaos injection, and assertions — and Sentinel runs everything for you.

This is especially useful for:

  • Non-Python team members who want to contribute tests
  • CI/CD pipelines where you want a declarative test suite
  • Regression testing with baselines (see Baselines)

The SentinelScenario Dataclass

Under the hood, a YAML file maps to a SentinelScenario (also aliased as TestScenario):

@dataclass
class SentinelScenario:
    id: str                                    # Unique scenario ID
    name: str                                  # Human-readable name
    description: str = ""                      # What this test verifies
    agent_config: Optional[AgentConfig] = None  # How to instantiate the agent
    environment: Optional[Environment] = None   # Pre-built environment (overrides env_config)
    env_config: Dict[str, Any] = {}            # Tool mock configuration
    chaos_config: Dict[str, Any] = {}          # Chaos injection rules
    task: str = ""                             # Task description for the agent
    assertions: List[Callable] = []            # Assertion functions to run
    timeout_seconds: int = 30                  # Max execution time
    tags: List[str] = []                       # Classification tags

YAML Example

Here's a complete scenario file:

# scenarios/refund_agent.yaml
id: refund-agent-001
name: "Refund agent handles timeout gracefully"
description: "Verify the refund agent retries on tool timeout and doesn't crash"
task: "Process a refund for order #12345"

# Mock tool configuration
env_config:
  tools:
    search:
      response:
        results:
          - "Refund Policy: 30-day window"
      latency_ms: 50
    database:
      response:
        user_id: 42
        status: active
        refund_eligible: true
    payment_api:
      response:
        refund_id: "REF-999"
        status: processed

# Chaos injection
chaos_config:
  tool_failures:
    - tool: search
      failure_rate: 0.3
      error_type: timeout

# Assertions (function names from sentinel.assertions)
assertions:
  - assert_tool_called
  - assert_no_tool_errors
  - assert_max_steps
  - assert_tool_called_in_order

# Agent configuration
agent_config:
  model: gpt-4
  tools:
    - search
    - database
    - payment_api

# Metadata
tags:
  - refund
  - resilience
  - critical
timeout_seconds: 60

Running Scenarios with the CLI

Run a specific scenario file

sentinel run --path scenarios/refund_agent.yaml

Run with verbose output

sentinel run --path scenarios/refund_agent.yaml --verbose

List all discovered scenarios

sentinel list

This discovers all scenario files in your project and prints their IDs, names, and tags.

Show details for a specific scenario

sentinel info refund-agent-001

Prints the full scenario config: task, tags, timeout, assertion count, and more.

Run all scenarios

sentinel run --all --verbose

JSON output for CI

sentinel run --all --json-output

Outputs structured JSON with pass/fail status, assertion results, and timing — perfect for piping into other tools or recording as a baseline.

The ScenarioRunner API

For programmatic use, use ScenarioRunner directly:

from sentinel.runner import ScenarioRunner, SentinelScenario, AgentConfig

# Define a scenario in Python
scenario = SentinelScenario(
    id="search-basic-001",
    name="Agent searches and returns results",
    task="Search for return policy information",
    env_config={
        "tools": {
            "search": {
                "response": {"results": ["Return policy: 30 days"]},
                "latency_ms": 25,
            }
        }
    },
    assertions=[],  # Add assertion callables here
    tags=["search", "smoke"],
)

# Create a runner and execute
runner = ScenarioRunner()
result = runner.run(scenario, agent_fn=my_agent_function)

print(f"Passed: {result.passed}")
print(f"Duration: {result.duration_ms:.0f}ms")
print(f"Summary: {result.summary}")

Running Multiple Scenarios

scenarios = [scenario_1, scenario_2, scenario_3]
runner = ScenarioRunner()
results = runner.run_batch(scenarios, agent_fn=my_agent_function)

for r in results:
    print(r.summary)
ℹ️ Agent function signature The agent_fn must accept three keyword arguments: task (str), env (Environment), and trace (AgentTrace). Your agent receives the mock environment and a trace to record its actions against.

Environment Config Format

The env_config.tools section maps tool names to their mock configuration:

FieldTypeDescription
responseanyStatic response returned by the tool
latency_msfloatSimulated response latency in milliseconds
error_probabilityfloatProbability of random errors (0.0–1.0)
