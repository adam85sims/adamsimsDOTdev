LangChain Adapter

Test real LangChain agents with Sentinel's mock tools, traces, and chaos injection.

Installation

Install the LangChain adapter as an optional extra:

pip install sentinel[langchain]

This pulls in langchain-core (≥ 0.2) for BaseTool, Runnable, and tool invocation protocols. If you skip the extra, you can still use the adapter for standalone mock-only testing without the framework.

SentinelToolAdapter

The core building block. SentinelToolAdapter wraps a LangChain BaseTool with a Sentinel MockTool, intercepting every invocation and recording it into an AgentTrace.

from sentinel.adapters.langchain import SentinelToolAdapter
from sentinel.env import MockTool
from sentinel.models import AgentTrace

trace = AgentTrace()

# Wrap a real LangChain tool with a mock
adapter = SentinelToolAdapter(
    mock=MockTool("search", response=SEARCH_RESULTS),
    trace=trace,
    base_tool=real_langchain_tool,  # original BaseTool (optional)
)

# Invoke it — delegates to the mock, records into trace
result = adapter.invoke(input={"query": "refund policy"})
print(f"Mock response: {result}")

Parameters

  • mock — A sentinel.env.MockTool providing canned responses, latency simulation, and error injection.
  • trace — An AgentTrace instance that records every tool call with arguments, results, duration, and errors.
  • base_tool — (Optional) The original LangChain BaseTool. If provided, the adapter preserves the tool's name, description, and args schema. If None, operates as a standalone callable proxy.

What Gets Preserved

✅ Preserved Tool name, description, argument schema, and invocation interface (invoke(), __call__()). The adapter is drop-in compatible with LangChain's tool protocol.

What Gets Replaced

⚠️ Replaced Execution → the mock tool provides the response instead of calling the real implementation. All other behavior (input parsing, validation) is preserved when a base_tool is provided.

Option 1: Wrapping Individual Tools

Wrap specific tools one at a time. Good when you want fine-grained control over which tools are mocked:

from sentinel.adapters.langchain import SentinelToolAdapter
from sentinel.env import MockTool
from sentinel.models import AgentTrace

trace = AgentTrace()

# Create mock tools
mock_search = MockTool("search", response={"results": ["doc1", "doc2"]})
mock_email = MockTool("email", response="sent")

# Wrap individual LangChain tools
search_adapter = SentinelToolAdapter(
    mock=mock_search, trace=trace, base_tool=langchain_search_tool
)
email_adapter = SentinelToolAdapter(
    mock=mock_email, trace=trace, base_tool=langchain_email_tool
)

# Wire into your agent
agent = create_react_agent(
    model,
    tools=[search_adapter, email_adapter],
    prompt=prompt,
)
result = agent.invoke({"messages": [...]})

# Inspect what happened
print(f"Search called {search_adapter.mock.call_count} times")
print(f"Email called {email_adapter.mock.call_count} times")
print(f"Trace has {trace.total_tool_calls} total calls")

Option 2: Wrapping the Entire Agent

Use wrap_agent() to wrap an entire LangChain agent at once. This creates an AgentWrapper that intercepts all tool calls through a tool_map:

from sentinel.adapters.langchain import wrap_agent
from sentinel.env import MockTool
from sentinel.models import AgentTrace

trace = AgentTrace()

# Define mock responses for every tool the agent uses
tool_map = {
    "search": MockTool("search", response={"results": ["doc1"]}),
    "email": MockTool("email", response="sent"),
    "database": MockTool("database", response={"rows": 42}),
}

# Wrap the entire agent
wrapped = wrap_agent(
    agent=my_langchain_agent,
    tool_map=tool_map,
    trace=trace,
)

# Use it exactly like the original agent
result = wrapped.invoke({"messages": [...]})

# Access individual adapters
search_adapter = wrapped.adapters["search"]
print(f"Search called {search_adapter.mock.call_count} times")

# Access the mock tool directly
search_mock = wrapped.get_mock("search")
print(f"Search mock responses: {search_mock.responses}")

Integration with Chaos Injectors

Combine adapters with Sentinel's chaos layer to test agent resilience:

from sentinel.adapters.langchain import wrap_agent
from sentinel.chaos import inject_failures
from sentinel.env import MockTool
from sentinel.models import AgentTrace

trace = AgentTrace()

# Create a mock tool, then inject chaos
search_mock = MockTool("search", response={"results": ["doc1"]})
inject_failures(search_mock, failure_rate=0.3, error_type="TimeoutError")

# Wrap agent with the chaotic mock
wrapped = wrap_agent(
    agent=my_agent,
    tool_map={"search": search_mock},
    trace=trace,
)

# Run multiple times to observe behavior under failure
for i in range(10):
    result = wrapped.invoke({"messages": [...]})
    # Agent should retry or degrade gracefully

# Check how many calls succeeded vs failed
successes = sum(1 for tc in trace.tool_calls if tc.error is None)
failures = sum(1 for tc in trace.tool_calls if tc.error is not None)
print(f"Successes: {successes}, Failures: {failures}")
