Quickstart

Get started with Sentinel in 5 minutes

Prerequisites

  • Python 3.9 or later
  • Git installed
  • A terminal or command prompt

Installation

Install Sentinel directly from GitHub:

pip install git+https://github.com/adam85sims/Sentinel.git
📦 What gets installed? Sentinel's core library has zero external dependencies. Framework adapters (LangChain, CrewAI, OpenAI SDK) are optional extras you install only if you need them.

To verify the installation succeeded:

python -c "import sentinel; print(f'Sentinel {sentinel.__version__} installed successfully')"

Your First Test

Create a file called test_quickstart.py and paste this code:

from sentinel.env import MockTool, EnvironmentBuilder
from sentinel.models import AgentTrace
from sentinel.assertions import assert_tool_called, assert_no_tool_errors

# 1. Create a mock environment with a search tool
env = (EnvironmentBuilder()
       .mock_tool("search", response={"results": ["Sentinel docs", "Quickstart guide"]})
       .build())

# 2. Record what your agent does
trace = AgentTrace()

# 3. Simulate your agent calling the tool
#    (Replace this with your actual agent's code)
result = env.tools["search"](query="how to test agents")
print(f"Tool returned: {result}")

# 4. Assert that the agent behaved correctly
assert_tool_called(trace, "search")
assert_no_tool_errors(trace)

print("All assertions passed!")

Run the test:

python test_quickstart.py

You should see output like this:

Tool returned: {'results': ['Sentinel docs', 'Quickstart guide']}
All assertions passed!

What Just Happened?

Let's break down each step:

1. Mock Environment

EnvironmentBuilder creates a controlled testing environment where all tools, APIs, and external services are mocked. Your agent never hits real infrastructure — it gets deterministic responses you define.

2. Agent Trace

AgentTrace records every tool call, error, and state change your agent makes. Think of it as a flight recorder for agent behavior. This is what Sentinel asserts against.

3. Tool Execution

env.tools["search"] calls the mock tool and logs the call to the trace. In production, this would be your agent calling a real search API — but here it's instant and deterministic.

4. Behavioral Assertions

assert_tool_called and assert_no_tool_errors verify properties about agent behavior — not just output values. They answer questions like: Did the agent call the right tools? Did it encounter errors? Did it recover gracefully?

✅ Key Insight Sentinel tests what agents DO, not just what they SAY. Traditional testing checks if the output matches an expected string. Sentinel checks if the agent's behavior — tool calls, error handling, state management — matches your specifications.

Adding Chaos (Optional)

Want to see what happens when things go wrong? Sentinel can inject failures into your mock tools:

from sentinel.chaos import inject_failures
from sentinel.env import EnvironmentBuilder

env = (EnvironmentBuilder()
       .mock_tool("search", response={"results": []})
       .build())

# Inject a 30% failure rate into the search tool
inject_failures(env, tool="search", failure_rate=0.3)

# Your agent now has to handle random tool failures
# This is where you discover if it retries, degrades gracefully, or crashes
