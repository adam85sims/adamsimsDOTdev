Baselines

Record, compare, and detect regressions in agent behavior.

What Are Baselines?

A baseline is a snapshot of your agent's test results at a point in time. You record a baseline when everything passes, then compare future runs against it to detect regressions — scenarios that used to pass but now fail.

Baselines are stored as JSON files in your project under .sentinel/baselines/, making them committable to version control.

Directory Structure

.sentinel/
  baselines/
    v1.2.3/
      metadata.json          # Label, timestamp, git info, tags
      results.json           # Serialized SentinelResult list
    nightly-abc1234/
      metadata.json
      results.json

Each baseline is a directory named by its label, containing two files:

  • metadata.json — label, timestamp, git SHA, branch, tags, scenario count, pass/fail counts
  • results.json — full serialized test results with traces, assertions, and timing

BaselineMetadata

Every baseline stores metadata alongside its results:

@dataclass
class BaselineMetadata:
    label: str                    # Human-readable ID (e.g. "v1.2.3")
    timestamp: float              # When it was recorded
    git_sha: str = ""             # Git commit SHA (auto-detected)
    git_branch: str = ""          # Git branch (auto-detected)
    tags: List[str] = []          # Classification tags
    description: str = ""         # What this baseline represents
    scenario_count: int = 0       # Number of scenarios in the baseline
    pass_count: int = 0           # Scenarios that passed
    fail_count: int = 0           # Scenarios that failed

Recording a Baseline

With Python

from sentinel.baseline import record_baseline

# After running your scenarios
path = record_baseline(
    results=results,              # List[SentinelResult]
    label="v1.2.3",              # Human-readable label
    tags=["release", "ci"],       # Optional classification tags
    description="Release v1.2.3 — all tests passing",
    git_sha="abc1234def5678",     # Auto-detected if omitted
    git_branch="main",            # Auto-detected if omitted
)
print(f"Baseline recorded at {path}")

With the CLI

First run your tests with JSON output, then pipe the results to record a baseline:

# Run tests and pipe to baseline
sentinel run --all --json-output | sentinel baseline record v1.2.3

# Or save to file first
sentinel run --all --json-output > results.json
sentinel baseline record v1.2.3 --path results.json

# With tags and description
sentinel baseline record v1.2.3 --tag release --tag ci --description "Pre-release snapshot"
ℹ️ Overwrite behavior If a baseline with the same label already exists, it's overwritten. This is intentional — updating a baseline is an explicit action, not silent data loss.

Loading a Baseline

from sentinel.baseline import load_baseline

metadata, results = load_baseline("v1.2.3")
print(f"Recorded: {metadata.timestamp}")
print(f"Git: {metadata.git_sha[:8]} ({metadata.git_branch})")
print(f"Tags: {metadata.tags}")
print(f"Results: {metadata.scenario_count} scenarios "
      f"({metadata.pass_count} pass, {metadata.fail_count} fail)")

# Use the results for comparison
for r in results:
    print(f"  [{r.passed}] {r.scenario_name}")

Listing Baselines

from sentinel.baseline import list_baselines

labels = list_baselines()  # Newest first
for label in labels:
    print(f"  {label}")

# CLI
sentinel baseline list

Deleting a Baseline

from sentinel.baseline import delete_baseline

deleted = delete_baseline("v1.2.3")
print(f"Deleted: {deleted}")  # True if found, False if not

# CLI
sentinel baseline delete v1.2.3

Comparing Baselines

Use build_regression_report to compare baseline results against current results and identify regressions, fixes, and new scenarios:

from sentinel.baseline import load_baseline
from sentinel.reporting import build_regression_report

# Load baseline
_, baseline_results = load_baseline("v1.2.3")

# current_results from your latest test run
report = build_regression_report(
    baseline_results=baseline_results,
    current_results=current_results,
    baseline_label="v1.2.3",
    current_label="current",
)

# Check for regressions
print(f"Verdict: {report.verdict}")     # PASS, FAIL, or WARN
print(f"Summary: {report.summary}")

if report.has_regressions:
    print("REGRESSIONS:")
    for d in report.regressions:
        print(f"  ✗ {d.scenario_name}: {d.new_failures}")

if report.fixes:
    print("FIXES:")
    for d in report.fixes:
        print(f"  ✓ {d.scenario_name}: {d.fixed_assertions}")

CLI Diff

# Compare two baselines
sentinel diff v1.2.3 v1.3.0

# JSON output
sentinel diff v1.2.3 v1.3.0 --json-output

Generating a Regression Report

Generate an HTML or JUnit XML report from a baseline:

# CLI
sentinel report --baseline v1.2.3 --format html --output ./reports
sentinel report --baseline v1.2.3 --format junit --output ./reports
sentinel report --baseline v1.2.3 --format both --output ./reports

# Python
from sentinel.reporting import generate_html_report, build_regression_report

report = build_regression_report(baseline_results, current_results)
html = generate_html_report(report)
with open("regression-report.html", "w") as f:
    f.write(html)
✅ Regression verdict The RegressionReport.verdict property returns:
  • "PASS" — no regressions, all scenarios still passing or new scenarios added
  • "FAIL" — regressions detected (scenarios that went from pass → fail)
  • "WARN" — no new regressions, but some scenarios were already failing
