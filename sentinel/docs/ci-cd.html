CI/CD Integration

Run Sentinel in GitHub Actions, GitLab CI, and more.

Overview

Sentinel integrates naturally into CI/CD pipelines. Run behavioral tests on every push, generate JUnit XML reports for CI dashboards, and detect regressions automatically with baselines.

GitHub Actions

Basic Workflow

Create .github/workflows/sentinel-tests.yml:

name: Sentinel Behavioral Tests

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  sentinel-tests:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.11"

      - name: Install Sentinel
        run: pip install git+https://github.com/adam85sims/Sentinel.git

      - name: Run Sentinel scenarios
        run: sentinel run --all --verbose

      - name: Generate JUnit XML report
        run: |
          sentinel run --all --json-output > results.json
          python -c "
          import json
          from sentinel.reporting import generate_junit_xml
          from sentinel.runner import SentinelResult, SentinelAssertionResult
          from sentinel.models import AgentTrace

          # Load results from JSON
          with open('results.json') as f:
              data = json.load(f)

          # Deserialize into SentinelResult objects
          results = []
          for r in data['results']:
              results.append(SentinelResult(
                  scenario_id=r['scenario_id'],
                  scenario_name=r['scenario_name'],
                  passed=r['passed'],
                  duration_ms=r['duration_ms'],
                  error=r.get('error'),
              ))

          # Generate JUnit XML
          xml = generate_junit_xml(results, suite_name='sentinel')
          with open('sentinel-results.xml', 'w') as f:
              f.write(xml)
          "

      - name: Publish test results
        uses: dorny/test-reporter@v1
        if: always()
        with:
          name: Sentinel Tests
          path: sentinel-results.xml
          reporter: java-junit

Using YAML Scenarios (Recommended for CI)

The simplest approach: define scenarios in YAML and run them with the CLI. No Python serialization needed:

name: Sentinel Scenarios

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"

      - run: pip install git+https://github.com/adam85sims/Sentinel.git

      - name: Run all scenarios
        run: sentinel run --all --verbose

      - name: Run specific scenario
        run: sentinel run --path scenarios/critical/refund_agent.yaml --verbose

GitLab CI

Create .gitlab-ci.yml:

sentinel-tests:
  image: python:3.11
  stage: test
  script:
    - pip install git+https://github.com/adam85sims/Sentinel.git
    - sentinel run --all --verbose
    - sentinel run --all --json-output > sentinel-results.json
  artifacts:
    when: always
    paths:
      - sentinel-results.json
    reports:
      junit: sentinel-results.xml
  rules:
    - if: $CI_MERGE_REQUEST_ID
    - if: $CI_COMMIT_BRANCH == "main"

For JUnit XML output in GitLab, add a step to generate the XML file before the job completes:

  script:
    - pip install git+https://github.com/adam85sims/Sentinel.git
    - sentinel run --all --json-output > sentinel-results.json
    - python -c "
        import json;
        from sentinel.reporting import generate_junit_xml;
        from sentinel.runner import SentinelResult;
        data = json.load(open('sentinel-results.json'));
        results = [SentinelResult(scenario_id=r['scenario_id'], scenario_name=r['scenario_name'], passed=r['passed'], duration_ms=r['duration_ms']) for r in data['results']];
        open('sentinel-results.xml', 'w').write(generate_junit_xml(results))
        "

Generating JUnit XML

Sentinel includes generate_junit_xml() to produce JUnit-compatible XML from test results. This works with any CI system that supports JUnit (GitHub Actions, GitLab CI, Jenkins, CircleCI, etc.).

from sentinel.reporting import generate_junit_xml

# After running scenarios
xml_output = generate_junit_xml(results, suite_name="sentinel-regression")

# Write to file
with open("sentinel-results.xml", "w") as f:
    f.write(xml_output)

The generated XML includes:

  • One <testcase> per scenario
  • Failure details with assertion names and error messages
  • Stdout with trace summary (tools called, steps, duration)
  • Total timing per scenario and for the suite

Exit Codes

The sentinel run command uses standard exit codes for CI integration:

Exit CodeMeaning
0All scenarios passed
1One or more scenarios failed (or agent crashed)

This means you can use Sentinel directly in CI without extra wrapper scripts — the exit code tells your pipeline whether to pass or fail the build.

Baseline-Driven Regression Detection

For advanced CI workflows, record a baseline on your main branch and compare every PR against it:

name: Regression Detection

on: pull_request

jobs:
  detect-regressions:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0  # Need full history for baseline loading

      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"

      - run: pip install git+https://github.com/adam85sims/Sentinel.git

      - name: Run current tests
        run: sentinel run --all --json-output > current-results.json

      - name: Detect regressions
        run: python -c "
          import json
          from sentinel.baseline import load_baseline
          from sentinel.reporting import build_regression_report

          # Load baseline from main branch
          _, baseline_results = load_baseline('main-latest')

          # Load current results
          with open('current-results.json') as f:
              current_data = json.load(f)

          # ... deserialize current_results ...

          report = build_regression_report(baseline_results, current_results)
          print(report.summary)

          if report.has_regressions:
              print('REGRESSIONS DETECTED')
              exit(1)
          else:
              print('No regressions')
          "
💡 Pro tip Store your baseline in a dedicated branch or use .sentinel/baselines/ committed to your repo. For PR-based workflows, you can also use the baseline record command on merge to main to keep the baseline fresh.
ℹ️ Non-deterministic agents If your agent uses a non-deterministic LLM, consider running scenarios multiple times or using chaos injection to test resilience rather than exact output matching. Behavioral assertions (tool calls, step counts) are more stable than output assertions for LLM-based agents.
